# OOP (객체지향이란)

## OOP 의 이해

### 1. OOP(Object Oriented Programmimg)
---

OOP란 객체지향으로 실세계에서 인지 할 수 있는 object(객체)의 개념을 가상세계에 적용한 것을 말한다.

실제 세계를 가상세계에 구현하고자 객체지향언어가 만들어졌다.
대표적인 예로 Java, Javascript, C++, C# 등이 있고 반대로 절차지향형인 C언어 가 존재한다.

객체지향이라하면 캡슐화(Encapsulation), 추상화(abstraction), 다형성(Ploymorphism) , 상속성(Inheritance)이 떠올라야한다.

초반에는 많이 사용되지는 않았지만 95년 java가 출시된 이후로 객체지향언어가 프로그래밍의 새로운 패러다임으로 자리매김을 했다.

객체지향언어의 장점은 크게 3가지로 코드의 재사용성과 관리가 용이, 신뢰성이 높은 프로그래밍을 가능하게해주는 장점을 가지고있다.

객체지향은 리얼월드를 가상월드로 옴기는 여러가지 여러가지 방법론중에 제일 많이 선택을 받는 방법론이다.

객체지향은 쉽게 부품을 나누고 관리하는 개념이다.

### 2. Object(객체)
---

객체란 사전적인 정의로는 '실제로 존재하는것'으로 물리적으로 존재하거나 인지가   
가능한 모든것을 객체라고 표현이 가능하다.

객체는 속성과 동작으로 구성이 되어 있는데 쉽게 말해서 이렇게 표현이 가능하다.  
ex) 사람이란 객체가 있으면 속성으로는 이름, 나이, 성별 등 / 동작으로는 걷다, 뛰다, 멈춰서다 등 이 있다.  

우리가 배우는 Java에서는 속성을 field,기능을 function 라고 부르는데 속성 보다는  
'멤버변수(member variable)'를 기능 보다는  '메소드(method)'를 주로 사용한다.

    객체 = 식별성 + 상태 +  행위

### 3. Class
---

클래스란 객체의 속성와 동작이 어떻게 만들지를 결정한 설계도이다.
제품을 만들때도 설계도를 잘 만들어놓으면 다시 만들경우 손쉽게 만들수가 있다.

객체 또한 class를 잘 만들어놓으면 객체를 생성 할 시 손쉽게 생성 할 수 있게된다.

즉 클래스는 객체를 정의해 놓은것이고, 용도로는 객체를 생성 할 때 사용이 가능하다.

Java에서는 기본적으로 프로그래밍에 유용한 class를 제공하여준다.  
클래스로부터 객체를 만드는 과정을 인스턴스화라고 하며 클래스로부터 만들어진 객체를 인스턴스라고도 부른다.


    class의 분류

      	App class (실체클래스?) /  Beans (추상적 클래스)
      - main(실행수단)이 꼭 필요  - R/W - RealWorld (상태 , 행위, 식별성)

### 4. abstraction(추상화)
---

추상화란 말 그대로 존재하지 않은 추상적인 것이다.

실체클래스는 객체를 생성 할 수 있을 정도로 구체화가 되어 있는 클래스라면,  
추상 클래스는 실체가 없는 추상적인 부분만 존재하여 객체를 생성 할 수 는 없다.   

추상 클래스와 실체클래스는 상속관계에 있다.  
말하자면 추상 클래스는 실체클래스를 용이하게 쓰기 위한 클래스 폼이라고 생각하면 쉽다.  

A라는 자동차 실체클래스 객체를 만들고 B라는 자동차를 추가 한다면 A클래스를 복사 붙여넣기 후 파일을 일일히 다 수정하여야 한다.  
하지만 자동차에 공통적으로 들어가는 바퀴, 오일, 핸들, 백미러, 크기 등 공통되는 상태(Field)나 행동(method)를  
모아 클래스로 만들어 추가 변경시 편리함을 준다.

예를 들어 '자동차 모닝 = new 자동차();'를 메모리 할당 하면 '자동차.크기'시 모닝의 저장된 '소형'이 출력된다.  

이것을 추상 클래스로 출력한다면 '모닝.set핸들("중형");' 시 중형으로 셋팅이되어 출력이 될것이다.  

또 '자동차 벤츠 = new 자동차();' 로 메모리를 할당하여 '벤츠.set크기("중형");' 으로 셋팅하여 출력이 가능하다.  

정리하자면 하나의 추상클래스로 공통된 여러가지의 정보를 편리하게 정리할 수 있게된다.

클래스의 타입으로 선언되었을 때 객체라고 부르고, 그 객체가 메모리에 할당되어 실제 사용될 때 인스턴스라고 부른다.  

객체는 현실 세계에 가깝고, 인스턴스는 소프트웨어 세계에 가깝다,  
객체는 '실체', 인스턴스는 '관계'에 초점을 맞춘다.  

### 5. Generalization(일반화)/(추상화 수준이 높다) , Inheritance(상속)
---

Generalization은 특수한 개념으로부터 공통된 개념을 묶는 것을 의미한다.  

Generic을 사용하면 코드를 간결하게 하며 개발자가 원하는 DataType으로 정의하여 재 사용성과 안정성을 제공한다.  

Inheritance은 은 말 그대로 자식클래스가 부모클래스로부터 무엇인가를 물려받는 것을 말한다.  

상속을 받는 방법으로는 extends를 선언하는데 자식 클래스 뒤에 extends 키워드를 사용하며 뒤에 부모클래스를 사용한다.  

자식은 하나의 부모에게만 부모는 여러 자식에게 상속이 가능하며 단일상속만 가능하다.  

상속(extends)는 일반적으로 공통된 Field나 method를 반복작업이 필요없이 나머지를 제외한 세부적인 것에 손쉽게 상속이 가능하다.  

Ex) 동물 이름, 사는곳 등... 이 일반적으로 공통된 것이다, 이것을 제외한 각각의 세부적인 귀의크기,  
지느러미의크기, 날개의크기 처럼  일반적이지 않은 것을 상속하여 이름, 사는곳을 상속받아 손쉽게 적용이 가능하다.  

PrimitiveType에서는 크키가 작것은 큰곳에 들어갈 수 있다고 정의되어있다.

하지만 Reference Type에서는 크기가 작은것이 큰곳에 들어간다는 개념은 있지만 실제로 값으로는 그 반대로 적용된다.  

Ex) 동물 <-- 사람 은 가능하다 / 동물 (이름, 사는곳) <-- 사람 (이름, 사는곳, 직업 ) 과 같이 
동물보다 사람이 포함하고있는 데이터가 더 많다.

### 6. Specialization(구체적)
---

추상화 상태를 상속을 통하여 좀 더 구체적인 형태로 발전시키는 과정 <-----> 반대로 추상화 수준이 높다.

name 과 age만 존재하는 Beans을 생성하여 job이 있는 Beans의 상속받고 school이 있는 Beans의 상속을 받아  
name과 age와 job, school이 존재하는 클래스를 만들수가 있게된다.  

결국 추상적으로만 존재하던 것에 다른 구체적인것을 이어붙여 구체적인 클래스로 표현이 가능해진다.

### 7. Realization(실체화)
---

추상화 형태의 객체를 상속하여 자신이 사용하고자하는 구현방법에 맞게 오버라이딩하는 것이다.

### 8. Association
---

객체 또는 클래스가 다른 객체 또는 클래스와 '어떤 의미의' 관계를 가질 수 있다.

### 9. dependency
---

또 다른 말로는 의존성이라고도 한다.  
다른 클래스에서 어떤 기능이 필요시에만, 클래스를 선언하여 사용하는 것이다.  
해당 클래스는 한번 사용 후 사라진다.  

쉽게 말하면 어느것에 의존하는것을(커플링 관계) 의존성이라하며   
나중에 배울 IOC에서 의존성에 대해 다뤄볼 수 있다.

ex) 병원에서 전산시스템과 행정시스템처럼 어떤 시스템을 작업하냐에 따라 상태, 행동, 식별성이 바뀐다.  (디커플링)

### 10. Polymorphism(다형성)
---

다형성은 OOP에서 매우 중요한 개념이다.

다형성은 상속을 통하여 기능을 확장시키던가 변경을 하는것을 말한다.  
하나의 클래스에 상속을 통하여서 목적과 용도에 맞게 결과를 출력 할 수 있다.  
다형성을 통해 코드의 재사용성과 관리가 용이하며, 신뢰성이 높은 프로그래밍을 할 수 있게 도와준다.  

다형성에는 오버라이딩(Overriding)과 오버로딩(Overloading)이 있다.  

Overriding은 클래스를 상속받아 만들어진 메소드를 재창조해서 사용하는 것을 말한다.  
Overloading은 하나의 클래스에 용도에 맞게 각각 다른 값을 출력 받게 구현하는 것을 말한다.  

### 11. Encapsulation / 절차은닉 / 정보은닉 (캡슐화)
---
캡슐화란 하나의 객체에 그 객체를 특정한 목적을 위해 필요한 변수나 클래스로 묶는 것을 의미한다.  

말 그대로 캡슐화는 중요한 데이터를 보존,보호 하기 위해서 사용한다.  
캡슐화를 하는 방법은 'private'라는 생성자를 사용하여 변수를 생성한다.

접근 방법으로는 setter와 getter 메소드가 존재하며 이를 사용하여 변수의 값을 조회 /변경이 가능하다.  

**private**
- setter (set) - 값을 넣어준다 (입력)  
- getter (get) - 값을 불러온다 (출력)  
- get(set)Xxxx() 로 사용하게된다.  
- 이것은 java개발자끼리의 묵시적으로 지키는 룰이다.

**추상화와 다형성의 관계**

추상화는 공통적으로 일반화된 상태나 행동을 하나의 클래스로 정리한것이다( 폼, 틀 ?)  
상속이란 추상화를 통해 만든 클래스를 더 구체적으로 구현할 때 클래스의 클래스를 만들며 중복되는 항목이 있을것이다.  

중복을 방지하기 위하여 상속을 사용하는데 상속을 사용시 상위클래스(부모클래스)의 있는 내용을 상속받아  
하위클래스(자식클래스)는 추가되는 항목만 작성하면 될 것이다.  

다형성은 이러한 추상화작업으로 만들어진 클래스를 통하여 상속과 여러가지 상황에 맞는 Type를 생성해내는것을 말한다. 

크게 오버라이딩이란 말그대로 '올라탄다'의 의미로 상속을 받아 만들어진 메소드를 재창조해서 사용하는 것을 말한다.  

오버로딩이란 하나의 클래스를 용도에 맞게 각각 다른 값을 출력 받게 구현하는 것을 말한다.
